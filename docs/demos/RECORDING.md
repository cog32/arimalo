# Recording the README demos

The screencasts embedded in the README are generated by
[`scripts/record_demos.mjs`](../../scripts/record_demos.mjs). It drives the real
Tauri app through three flows, records the app window with macOS `screencapture`,
and encodes muted MP4 + WebM (plus a poster frame) into this folder.

```sh
npm run record:demos                       # all three flows
npm run record:demos -- --flow manual-txn  # one flow
npm run record:demos -- --flow getting-started
npm run record:demos -- --flow import-csv
```

Flows: `getting-started` (vault picker + add account), `import-csv`, `manual-txn`.
Useful flags: `--keep-mov` (keep the raw capture), `--no-record` (drive only, no
video), `--no-encode` (keep the raw `.mov`), `--pace <ms>`, `--width <px>`.

## Prerequisites

- **macOS** (the script uses `screencapture` and `osascript`).
- **App binary built with the webdriver feature** — run `./scripts/run_debug_mcp.sh`
  once. The script looks for `~/.cache/arimalo-target-mcp/debug/arimalo-covid`
  (falling back to `src-tauri/target/debug/arimalo-covid`).
- **`tauri-wd`** on `PATH` or in `~/.cargo/bin` (`cargo install tauri-webdriver-automation`).
- **`ffmpeg` / `ffprobe`** (`brew install ffmpeg`).
- `selenium-webdriver` (already a dev dependency).

## Required macOS permissions (TCC)

The terminal/app that *runs* the script needs two permissions in
**System Settings → Privacy & Security**:

| Permission | Why | Which flows |
|------------|-----|-------------|
| **Screen Recording** | `screencapture` records the app window | all |
| **Accessibility** | `osascript` types into the native file/folder dialogs | `getting-started`, `import-csv` |

`manual-txn` is fully in-WebView and needs only Screen Recording. The other two
drive native OS dialogs via synthetic keystrokes, which silently do nothing
without Accessibility. Grant both to the app hosting your shell (e.g.
**Terminal**), then re-run.

## Safety

- The script never touches your real vault: it moves
  `~/Library/Application Support/com.cog32.arimalocovid/config.json` aside,
  points the app at a throwaway temp vault, and restores your config afterwards
  (including on crash/Ctrl-C). Demo data is seeded into the temp vault only.
- It takes exclusive ownership of `tauri-wd` on port 4444 for the duration
  (stop any MCP/`tauri-wd` session first).
